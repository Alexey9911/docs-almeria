## Architecture Security

### Client-Side Security

All sensitive operations happen in your browser:

```typescript
// Private keys never leave the browser
const transaction = new Transaction().add(instruction);
const signedTransaction = await wallet.signTransaction(transaction);

// Only signed transactions are sent to our services
const result = await fetch('/api/broadcast', {
  method: 'POST',
  body: JSON.stringify({ 
    signedTransaction: signedTransaction.serialize() 
  })
});
```

**Security Properties:**
- Private keys remain in wallet browser extension
- Transactions are signed locally before transmission
- Our services only see signed, committed transactions
- No ability to access or move user funds

### Routing Layer Security

Our privacy routing infrastructure is designed with security-first principles:

<Steps>
  <Step title="Ephemeral Addresses">
    Each transaction uses a fresh, single-use deposit address generated with cryptographically secure randomness.
  </Step>
  <Step title="Time-Limited Operations">
    Deposit addresses expire after 24 hours. Funds sent to expired addresses are automatically refunded.
  </Step>
  <Step title="Atomic Processing">
    Routing operations complete atomically or fail completely. No partial states or stuck funds.
  </Step>
  <Step title="Monitoring & Alerts">
    Real-time monitoring of all routing operations with automated alerting for anomalies.
  </Step>
</Steps>

## Bug Bounty Program

We maintain an active bug bounty program to incentivize security research:

### Scope

**In Scope:**
- Web application (app.unipay.com)
- API endpoints (api.unipay.com)
- Client-side JavaScript code
- Smart contract interactions
- Privacy model vulnerabilities

**Out of Scope:**
- Social engineering attacks
- Physical security
- Third-party services (wallets, RPC providers)
- Denial of service attacks

### Rewards

| Severity | Reward Range | Examples |
|----------|--------------|----------|
| **Critical** | $10,000 - $50,000 | Fund theft, private key extraction |
| **High** | $5,000 - $15,000 | Transaction manipulation, privacy bypass |
| **Medium** | $1,000 - $5,000 | Information disclosure, DoS |
| **Low** | $100 - $1,000 | Minor information leaks, UX issues |

### How to Report

1. **Email**: security@unipay.com (PGP key available)
2. **Include**: Detailed reproduction steps and impact assessment
3. **Response**: We respond within 48 hours on business days
4. **Timeline**: 90 days for resolution before public disclosure

### For Developers

```typescript
// Always validate inputs
const validateAddress = (address: string): boolean => {
  try {
    new PublicKey(address);
    return true;
  } catch {
    return false;
  }
};

// Use proper error handling
try {
  const transaction = await buildTransaction(params);
  const signature = await sendTransaction(transaction);
  return { success: true, signature };
} catch (error) {
  console.error('Transaction failed:', error);
  return { success: false, error: error.message };
}

// Implement proper session management
const sessionKey = localStorage.getItem('unipay_session_key_v1');
if (!sessionKey || isExpired(sessionKey)) {
  generateNewSession();
}
```

## Incident Response

### Response Team

Our security incident response team includes:
- **Security Lead**: Coordinates response and external communication

## Compliance & Regulations

### Regulatory Compliance

We work with legal experts to ensure compliance with applicable regulations:

- **AML/KYC**: No customer data collected, but routing layer maintains transaction logs
- **GDPR**: Minimal data collection, user control over session data
- **SOX**: Financial controls and audit trails for corporate users
- **Regional Laws**: Compliance with local privacy and financial regulations

### Data Handling

| Data Type | Storage | Retention | Access |
|-----------|---------|-----------|--------|
| **Transaction Logs** | Encrypted, routing layer | 7 years | Legal process only |
| **Session Keys** | Browser localStorage | User controlled | User only |
| **API Logs** | Encrypted, no PII | 90 days | Internal security team |
| **Error Reports** | Anonymized | 30 days | Engineering team |
