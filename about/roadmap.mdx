## Q2 2026 - Scale & Optimize

### Enhanced Batch Processing

Batch sizes beyond 12 recipients via transaction splitting and lookup tables. Implement Solana lookup tables for address compression. Split large payrolls across multiple transactions automatically. Maintain single-signature UX through transaction chaining.

**Impact:** Support payrolls of 50+ recipients in wallet mode

### Deterministic Session Keys

Session-key derivation from connected wallet signature, eliminating backup files. Derive session keys from wallet signature over standard message. Eliminate need for manual backup/restore workflow.

**Impact:** Simplified onboarding, no lost session keys

## Q3 2026 - Privacy Enhancements

### Advanced Timing Obfuscation

Sophisticated batching and delay randomization to resist timing analysis. Implement variable-size batching windows. Add decoy transactions to obscure real transaction timing.

**Impact:** Stronger protection against timing correlation attacks

### Cross-Chain Privacy

Extend graph-break privacy to Ethereum and other EVM chains. Deploy routing infrastructure on Ethereum mainnet. Support ETH, USDC, USDT cross-chain private transfers.

**Impact:** Privacy for multi-chain users and cross-chain transactions

## Q4 2026 - Developer Platform

### SDK Release

TypeScript SDK for third-party wallet and dApp integration:

```typescript
import { UnipaySDK } from '@unipay/sdk';

const unipay = new UnipaySDK({
  apiKey: 'your-api-key',
  network: 'mainnet-beta'
});

await unipay.send({
  recipient: 'address',
  amount: 100,
  asset: 'USDC',
  mode: 'private'
});
```

**Integration Partners:** Phantom Wallet, Solflare, Backpack, Jupiter

### API Platform

REST API for institutional and programmatic access. Endpoints for quotes, transactions, batch operations, and status tracking.

## Q1 2027 - Advanced Features

### Shielded Pool Integration

Optional integration with production-grade Solana privacy primitives. Conditional on Solana shielded pool reaching production readiness and regulatory clarity.

**Technical Approach:** Hybrid model with user choice between privacy levels

### Compliance Tools

Advanced compliance and reporting features for institutional users. Automated transaction reporting, compliance dashboard, integration with accounting software.

**Target Users:** Crypto-native businesses, DeFi protocols, institutional treasury management

## Long-Term Vision (2027+)

### Decentralized Routing

Transition from centralized routing layer to decentralized network. Validator-operated routing nodes with economic incentives and slashing conditions.

### Zero-Knowledge Proofs

Research and development of ZK-based privacy solutions. ZK-SNARKs for transaction privacy, recursive proof systems for scalability.
