## Core Principles

We ship small. We write the threat model before the feature. We treat user funds as uninsured and build accordingly. We do not own user state and we do not want to.

### Engineering Philosophy

<Steps>
  <Step title="Security First">
    Every feature begins with threat modeling. We assume adversarial conditions and build defensive systems.
  </Step>
  <Step title="Boring Technology">
    We prefer well-understood, battle-tested primitives over novel cryptography we cannot audit ourselves.
  </Step>
  <Step title="Stateless by Design">
    Our services hold no user state. Rebooting servers loses no user data because there is none to lose.
  </Step>
  <Step title="Build in Public">
    Open source where possible, transparent about limitations, honest about tradeoffs.
  </Step>
</Steps>
