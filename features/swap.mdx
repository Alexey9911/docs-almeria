# Swap

## Execution Modes

### Wallet Mode

Direct swap through on-chain aggregator:

<Steps>
  <Step title="Get Quote">
    Client queries on-chain aggregator for best available rate across all DEXs.
  </Step>
  <Step title="Build Transaction">
    Aggregator returns a versioned transaction with optimal routing.
  </Step>
  <Step title="Sign Locally">
    Your wallet signs the transaction in the browser.
  </Step>
  <Step title="Execute via Proxy">
    Transaction is broadcast through a stateless proxy service.
  </Step>
</Steps>

**Aggregator Integration:**
```typescript
// Quote request
const quote = await aggregator.getQuote({
  inputMint: SOL_MINT,
  outputMint: USDC_MINT,
  amount: inputAmount,
  slippageBps: slippageTolerance
});

// Transaction construction
const swapTransaction = await aggregator.getSwapTransaction({
  quote,
  userPublicKey: wallet.publicKey
});
```

### Private Mode

Anonymous swap through routing layer:

<Steps>
  <Step title="Request Private Quote">
    Quote the same pair through the routing layer for privacy-enhanced execution.
  </Step>
  <Step title="Deposit to Ephemeral Address">
    Send SOL to the provided temporary address.
  </Step>
  <Step title="Routing Layer Swap">
    Routing layer executes the swap using its own addresses.
  </Step>
  <Step title="Receive Output">
    You receive USDC/USDT from an address unrelated to your input wallet.
  </Step>
</Steps>

**Privacy Benefits:**
- Input SOL address is not linked to output stablecoin address
- Swap execution happens through routing layer's addresses
- Your wallet balance changes are not directly observable

## Slippage Protection

Configure your tolerance for price movement during execution:

| Setting | Tolerance | Use Case |
|---------|-----------|----------|
| **Auto** | Dynamic based on market conditions | Recommended for most users |
| **0.5%** | Low tolerance | Stable market conditions |
| **1%** | Medium tolerance | Normal market volatility |
| **3%** | High tolerance | High volatility periods |
| **Custom** | User-defined (bps resolution) | Advanced users |

```typescript
// Slippage calculation
const minOutputAmount = expectedOutput * (1 - slippageBps / 10000);

// Transaction will fail if actual output < minOutputAmount
```

<Warning>
Swaps that exceed your slippage tolerance are rejected before you sign the transaction. No funds are lost due to slippage protection.
</Warning>

Large swaps may experience price impact:

| Swap Size | Typical Impact |
|-----------|----------------|

## Advanced Features

### MEV Protection

All swaps include MEV (Maximum Extractable Value) protection:

- **Private mempools**: Transactions routed through protected pools
- **Atomic execution**: Swap completes in single transaction or fails
- **Front-running protection**: Orders not visible until execution

### Failed Transaction Handling

If a swap fails:

1. **Wallet Mode**: Transaction reverts, no funds lost, only network fees charged
2. **Private Mode**: Routing layer refunds input asset minus processing fee

## Fees

## Troubleshooting

### Common Issues

| Error | Cause | Solution |
|-------|-------|----------|
| "Insufficient liquidity" | Large swap size | Reduce amount or try later |
| "Slippage exceeded" | Price moved during execution | Increase slippage tolerance |
| "Quote expired" | Took too long to confirm | Refresh quote and retry |
| "Routing unavailable" | Private mode capacity | Use wallet mode or retry later |

### Performance Tips

- **Optimal timing**: Avoid high-volatility periods for better rates
- **Size optimization**: Consider multiple smaller swaps for large amounts
- **Slippage tuning**: Use Auto for most cases, custom only if needed

## Security Considerations

<Warning>
**Private Mode Disclosure**: While private mode breaks the on-chain link between your SOL and stablecoin addresses, the routing layer can recover this linkage if compelled by legal process. This is not cryptographic anonymity.
</Warning>

### Best Practices

- **Verify quotes**: Always review price impact and output amount
- **Check addresses**: Ensure you're swapping to the correct token
- **Monitor execution**: Watch transaction status until completion
- **Keep records**: Save transaction IDs for your records
