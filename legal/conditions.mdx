### Wallet Standard Compliance

All supported wallets must implement:
- Solana Wallet Standard interface
- Transaction signing capabilities
- Account connection/disconnection
- Network switching (mainnet-beta)

```typescript
// Required wallet capabilities
interface WalletCapabilities {
  connect(): Promise<PublicKey>;
  disconnect(): Promise<void>;
  signTransaction(transaction: Transaction): Promise<Transaction>;
  signAllTransactions(transactions: Transaction[]): Promise<Transaction[]>;
}
```

## 2. Transaction Limits

### Amount Limits

| Asset | Wallet Mode | Private Mode |
|-------|-------------|--------------|
| **SOL** | No limit | Routing layer minimum (varies) |
| **USDC** | No limit | Routing layer minimum (varies) |
| **USDT** | No limit | Routing layer minimum (varies) |

<Note>
Private flows are subject to routing layer minimums that vary per asset and change without notice. The client surfaces the minimum inline when an amount falls below it.
</Note>

### Batch Limits

| Operation | Wallet Mode | Private Mode |
|-----------|-------------|--------------|
| **Payroll Recipients** | 12 maximum | No limit |
| **Transaction Size** | 1232 bytes (Solana limit) | N/A |
| **Instructions** | ~20 per transaction | N/A |

**Wallet Mode Limitation:**
Wallet-mode payroll is capped at 12 recipients per signed transaction to stay within the Solana 1232-byte transaction limit.

**Private Mode Scaling:**
Private-mode payroll has no per-batch cap but scales linearly in routing-layer calls.

## 3. Network Requirements

- **Supported Network**: Solana mainnet-beta only
- **Devnet/Testnet**: Not exposed in the UI
- **Token Programs**: Classic Token Program and Token-2022 (balance display only)

### RPC Configuration

```typescript
// Default RPC endpoint
const RPC_ENDPOINT = process.env.NEXT_PUBLIC_SOLANA_RPC || 
                    'https://api.mainnet-beta.solana.com';

// Solana Labs public endpoint is blocked for reliability
if (RPC_ENDPOINT.includes('api.mainnet-beta.solana.com')) {
  throw new Error('Public endpoint blocked - use custom RPC');

```

### Custom RPC Usage

Users supplying their own RPC endpoint via `NEXT_PUBLIC_SOLANA_RPC` are responsible for:
- Staying within provider rate limits

- Monitoring for service degradation

## 4. Slippage and Pricing

### Slippage Tolerance

Available slippage settings for swaps:

| Setting | Tolerance | Recommended For |
|---------|-----------|-----------------|
| **Auto** | Dynamic (0.1-2%) | Most users |
| **0.5%** | Fixed 0.5% | Stable conditions |
| **1%** | Fixed 1% | Normal volatility |
| **3%** | Fixed 3% | High volatility |
| **Custom** | User-defined | Advanced users |

### Slippage Protection

<Warning>
Swaps that exceed your slippage tolerance are rejected before you sign the transaction. No funds are lost due to slippage protection, but you may need to adjust tolerance or retry.
</Warning>

```typescript
// Slippage calculation
const minOutputAmount = expectedOutput * (1 - slippageBps / 10000);

// Transaction fails if: actualOutput < minOutputAmount
```

## 5. Session Management

### Session Keys

The client generates a session keypair on first use:

```typescript
// Session key storage
const sessionKey = Keypair.generate();
localStorage.setItem('unipay_session_key_v1', 
  base64.encode(sessionKey.secretKey));
```

### Backup Responsibility

<Warning>
**Critical**: The session key lives only in your browser. Clearing site data without first exporting the recovery file results in permanent loss of session-scoped state. Funds custodied by your primary wallet are unaffected.
</Warning>

### Recovery Process

1. **Export**: Download recovery file (.txt format)
2. **Store Safely**: Keep backup file secure and accessible
3. **Import**: Upload recovery file to restore session
4. **Verify**: Check balance reconciliation after restore

## 6. Fee Structure

### Network Fees

All transactions include standard Solana network fees:
- **Base Fee**: ~0.000005 SOL per signature
- **Instruction Fee**: ~0.000005 SOL per instruction
- **Priority Fees**: Variable based on network congestion

### Service Fees

| Mode | Additional Fees |
|------|-----------------|
| **Wallet Mode** | None - network fees only |
| **Private Mode** | Routing layer fee (varies by asset) |

### Fee Calculation Examples

```typescript
// Wallet mode: SOL transfer
const fee = 0.000005; // Single signature

// Wallet mode: Payroll (5 recipients)
const fee = 0.000005 + (5 * 0.000005); // Base + instructions

// Private mode: Additional routing fee
const totalFee = networkFee + routingFee;
```

## 9. Error Handling

### Common Error Scenarios

| Error Type | Cause | User Action |
|------------|-------|-------------|
| **Insufficient Balance** | Not enough tokens + fees | Add funds or reduce amount |
| **Network Congestion** | High transaction volume | Retry with higher priority fee |
| **Slippage Exceeded** | Price movement during swap | Increase slippage tolerance |
| **Routing Unavailable** | Private mode capacity | Use wallet mode or retry later |

### Error Reporting

<Note>
Failure is loud: Quote failures, RPC errors, and routing errors surface on the same screen that caused them, with the machine-readable code shown below the human-readable message.
</Note>
